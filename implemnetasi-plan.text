Refactoring Plan: AiStudioSection.tsx (Industry Standard)
1. Ekstraksi Konstanta & Helper (Global Scope)

Pindahkan styleOptions, platformOptions, dan fungsi getPreviewUrl ke luar fungsi komponen utama (di bagian paling atas file).
Gunakan penamaan variabel Screaming Snake Case (contoh: STYLE_OPTIONS) untuk data statis agar sesuai standar profesional.

2. Konsolidasi State (State Grouping)

Gabungkan useState yang terpisah-pisah menjadi dua kelompok objek state utama:
formData: Mengelompokkan style, platforms, imagePreview, imageFile, dan base64.
uiState: Mengelompokkan state tampilan seperti isVisible, styleDropdownOpen, dan platformDropdownOpen.
Tujuannya untuk menyederhanakan update state dan meminimalkan jumlah baris deklarasi.

3. Implementasi Custom Hook untuk Logika (Separation of Concerns)

Pindahkan semua logika "otak" komponen ke dalam satu Custom Hook baru (misal: useAiStudioController).
Logika yang dipindah meliputi:
IntersectionObserver untuk animasi scroll.
handleImageUpload (pemrosesan file ke base64).
handleGenerate dan handleReset (interaksi dengan API dan localStorage).
Sinkronisasi awal data dari localStorage.
Komponen utama hanya akan menerima data siap pakai dan fungsi handler dari hook ini.

4. Perbaikan Inisialisasi State (Lazy Initialization)

Pastikan pembacaan localStorage hanya terjadi satu kali saat mount menggunakan fungsi anonim di dalam useState.
Hapus pemanggilan setState di dalam useEffect yang menyebabkan "Cascading Renders" (render dua kali).

5. Pembersihan JSX (Icon & Sub-Render)

Ekstrak semua tag <svg> yang panjang ke dalam komponen fungsi kecil (Functional Components) di bagian bawah file (contoh: const LoadingIcon = () => (...)).
Gunakan teknik Sub-Rendering (membuat fungsi render di dalam file yang sama) untuk bagian UI yang kompleks seperti bagian Dropdown atau Card Preview, agar struktur utama JSX di bagian return tetap bersih dan mudah dibaca.
